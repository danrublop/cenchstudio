# D3 Scene Rules

---

## CenchCharts Presets (preferred)

For standard charts, use the `generate_chart` tool instead of writing raw D3 code. It calls the CenchCharts library which is pre-loaded in every D3 scene.

### Available chart types

| chartType       | Data format                             | Notes                                                                     |
| --------------- | --------------------------------------- | ------------------------------------------------------------------------- |
| `bar`           | `[{label, value}]`                      | Vertical bars with axes, grid, value labels                               |
| `horizontalBar` | `[{label, value}]`                      | Horizontal bars                                                           |
| `line`          | `[{label, value}]`                      | Line chart with dots. Multi-series: `[{label, values: [{label, value}]}]` |
| `area`          | `[{label, value}]`                      | Area chart with line and fill                                             |
| `pie`           | `[{label, value}]`                      | Pie chart                                                                 |
| `donut`         | `[{label, value}]`                      | Donut with center total                                                   |
| `scatter`       | `[{label, value}]` or `[{x, y, size?}]` | Scatter plot                                                              |
| `number`        | `{value, label}`                        | Large KPI number display                                                  |
| `gauge`         | `{value, max}`                          | Gauge/meter arc                                                           |
| `stackedBar`    | `[{label, values: {key: num}}]`         | Stacked vertical bars                                                     |
| `groupedBar`    | `[{label, values: {key: num}}]`         | Grouped/clustered bars                                                    |

### Config options

```js
{
  title, subtitle,              // Chart title and subtitle
  xLabel, yLabel,               // Axis labels
  valueFormat: ",.0f",          // d3.format string
  valuePrefix: "$",             // Prepended to formatted values
  valueSuffix: "M",             // Appended to formatted values
  showValues: true,             // Show value labels on data elements
  showGrid: true,               // Show background gridlines
  showLegend: false,            // Show legend (auto for multi-series)
  colors: [...],                // Override colors (defaults to PALETTE)
  theme: "dark" | "light",      // Match scene background
  animationDuration: 1.2,       // Seconds for main data reveal
  staggerDelay: 0.08,           // Seconds between staggered elements
  countDuration: 1.0,           // Seconds for counting number animations
  plotBackground: "#ffffff",    // Optional fill behind marks inside the SVG; omit or "transparent" to see scene bg through
  chartPanelBackground: "transparent", // HTML container behind the chart (percent-positioned div); rgba for glass overlays
  chartPanelOpacity: 1,
  chartPanelBorderRadius: 12,
  chartPanelBoxShadow: "0 8px 24px rgba(0,0,0,0.12)",
}
```

`chartPanel*` keys are stripped before calling CenchCharts and applied in compiled `sceneCode` so charts can float over video, canvas motion backgrounds, or images without fusing to the scene background color.

Readability defaults (unless user explicitly requests otherwise):

- Include clear chart title and axis labels (for axis-based charts)
- Keep value labels/grid/legend enabled when they improve readability
- Use legible typography and sufficient contrast

### Animated mode

Set `animated: true` on the `generate_chart` tool to enable cinematic reveals:

- Title/subtitle fade in
- Axes and gridlines appear
- Data elements animate (bars grow from baseline, lines draw, pies sweep, numbers count up)
- Value labels count up from 0

All animations go onto `window.__tl` so they're seekable via the scrub bar.

### When to use raw D3 instead

Use `add_layer` with `layerType: 'd3'` only for visualizations that don't fit a preset:

- Network/force graphs
- Treemaps, sunburst, chord diagrams
- Custom/artistic data visualizations
- Heavily customized layouts

## Output format (raw D3 mode)

Output raw JSON only. No markdown fences. No explanation:

```json
{
  "styles": "<CSS string — no <style> tags>",
  "sceneCode": "<JavaScript using D3 v7 global>",
  "suggestedData": { ... }
}
```

The template injects `styles` into a `<style>` block, `sceneCode` into a `<script>`, and pre-populates `DATA` from `suggestedData`.

## Globals available in sceneCode

- `d3` — D3 v7 (loaded from CDN)
- `DATA` — pre-filled from `suggestedData`
- `WIDTH` (default 1920), `HEIGHT` (default 1080) — dimensions change based on the project's aspect ratio

## SVG setup

Always create the SVG this way:

```js
const svg = d3
  .select('#chart')
  .append('svg')
  .attr('viewBox', `0 0 ${WIDTH} ${HEIGHT}`)
  .attr('width', '100%')
  .attr('height', '100%')
```

### Common mistakes

**NEVER** set fixed pixel dimensions on the SVG:

```js
// ❌ WRONG — creates oversized SVG that overflows the container
.append('svg').attr('width', WIDTH).attr('height', HEIGHT)

// ✅ CORRECT — viewBox defines the coordinate space, 100% scales to fit
.append('svg')
  .attr('viewBox', `0 0 ${WIDTH} ${HEIGHT}`)
  .attr('width', '100%')
  .attr('height', '100%')
```

Using `attr('width', 1920)` creates a literal 1920px-wide SVG element that won't scale to fit its container. The `viewBox` + `100%` pattern lets the browser scale the SVG responsively while preserving the WIDTH×HEIGHT coordinate system for all internal positioning.

## Animation patterns

All enters must transition:

```js
.transition()
.duration(800)
.ease(d3.easeCubicOut)
.delay((d, i) => i * 100)
```

- Elements start at opacity 0, transition to 1
- Bars start at height 0, transition to full height (transform from bottom)
- Use `d3.easeCubicOut` for most transitions; `d3.easeElasticOut` for emphasis

## Font sizes

- Title: 48px bold
- Axis labels: 20px
- Data labels: 18px
- Annotations: 16px

---

## Chart types and patterns

**Bar chart:** use `scaleBand` for x, `scaleLinear` for y. Add axes with `axisBottom`/`axisLeft`. Animate bars with height 0 → full.

**Line chart:** use `line()` generator. Animate with `stroke-dasharray` / `stroke-dashoffset` trick.

**Pie/donut:** use `arc()` + `pie()`. Animate with `attrTween` on `d`.

**Scatter:** circles fade in with staggered delay.

## suggestedData

Provide a realistic dataset matching the prompt topic. Structure it so `DATA` is directly usable in `sceneCode` without transformation. Use real-world plausible numbers.

---

## Colors

Use the palette in your styles: `#181818, #121212, #e84545, #151515, #f0ece0`

- Bars/lines: `#e84545` (accent) or `#f0ece0` (light)
- Backgrounds: `#181818`
- Text: `#f0ece0`
- Axes/gridlines: `#333333` or `rgba(240,236,224,0.2)`

## HTML template

The D3 template is injected by the rendering system. For the file write, use:

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <style>
      * { margin: 0; padding: 0; box-sizing: border-box; }
      html, body { width: 100%; height: 100vh; overflow: hidden; background: BG_COLOR_HERE; }
      #chart { width: 100%; height: 100%; }
      STYLES_HERE
    </style>
  </head>
  <body>
    <div id="chart"></div>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/d3/7.9.0/d3.min.js"></script>
    <script>
      const DATA = D3_DATA_HERE
      const WIDTH = WIDTH_HERE,
        HEIGHT = HEIGHT_HERE
      window.__pause = () => {}
      window.__resume = () => {}
      SCENE_CODE_HERE
    </script>
  </body>
</html>
```

Replace `BG_COLOR_HERE`, `STYLES_HERE` (raw CSS, no `<style>` tags), `D3_DATA_HERE` (JSON-serialized suggestedData), and `SCENE_CODE_HERE` (the sceneCode string).
