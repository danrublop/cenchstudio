# Third-party audio — sources and licenses

This file records **upstream projects and licenses** for audio that may ship with Cench Studio (for example under `public/sfx-library/`). It is not legal advice; verify before commercial distribution.

## ZzFX (current bundled procedural library)

- **Source:** [KilledByAPixel/ZzFX](https://github.com/KilledByAPixel/ZzFX)
- **License:** MIT
- **Use here:** In-app synthesis and WAVs generated by `npm run sfx-library:zzfx` (`librarySource: "zzfx"` in `public/sfx-library/manifest.json`).

## SoLoud (optional imports — engine / assets)

- **Source:** [jarikomppa/soloud](https://github.com/jarikomppa/soloud)
- **License:** SoLoud proper is **zlib/libpng** (permissive; **commercial use is explicitly allowed**). The upstream `LICENSE` file also notes bundled third-party code with similarly liberal terms; see [SoLoud legal / manual](http://soloud-audio.com/legal.html) and the full text in the repo:  
  [github.com/jarikomppa/soloud/blob/master/LICENSE](https://github.com/jarikomppa/soloud/blob/master/LICENSE)

**zlib/libpng summary (SoLoud):** use for any purpose including commercial apps; do not misrepresent origin; mark altered sources; keep the license notice in source distributions.

**Using SoLoud in this project:** SoLoud is a **C/C++ engine** (not a JS npm package). Typical approaches are WASM/native bindings or exporting **audio files** from SoLoud-powered tools/demos. Any WAV/OGG you add from SoLoud (or generated with it) should be listed in `manifest.json` with:

- `license`: e.g. `zlib/libpng (SoLoud)` or as required by the specific file’s origin
- `librarySource`: `"soloud"` (see `lib/audio/sfx-local-manifest.ts`)

Add files under `public/sfx-library/{category}/` using a `soloud-` filename prefix so they stay git-tracked alongside the `zzfx-*.wav` exception in `.gitignore`.

## react-sounds (optional imports — curated SFX pack)

- **Source:** [e3ntity/react-sounds](https://github.com/e3ntity/react-sounds)
- **License:** **MIT** (explicitly allows commercial use; retain copyright notice in copies of the software — see the repo [LICENSE](https://github.com/e3ntity/react-sounds/blob/master/LICENSE)).
- **What it is:** A **React + Howler.js** library. Sounds are usually loaded from a **CDN** at runtime; the repo also documents **offline / self-host** via their CLI (`bin` / `scripts` in the package).

**Using it in Cench Studio:** We do **not** depend on `react-sounds` in the editor bundle today. The practical approach matches other static SFX:

1. **Self-host** clips (e.g. using the project’s documented download flow) into `public/sfx-library/{category}/`.
2. Name files with a **`react-sounds-` prefix** (e.g. `ui/react-sounds-button_1.mp3`) so `.gitignore` allows them.
3. Add manifest entries with `librarySource: "react-sounds"` and `license: "MIT (react-sounds)"` (or the exact notice you use for shipped copies).

That keeps **export, publish, and the render server** on same-origin URLs under `/sfx-library/`, instead of runtime CDN URLs inside scene HTML.

## Music Megathread (reference only — not a product catalog)

- **Source:** [MoonWalker440/Music-Megathread](https://github.com/MoonWalker440/Music-Megathread)
- **What it is:** A large **README of links** to music-related apps, sites, players, discovery tools, and similar resources. It is **not** a repository of downloadable tracks under one license, and it is **not** code or assets this app can “import” the way we bundle ZzFX WAVs or self-hosted SFX.
- **Licensing / commercial use:** Entries point to **many unrelated services and tools**. Some are FOSS; many are third-party commercial or **grey-area** (e.g. unofficial clients, downloaders). **Nothing in that list is automatically safe for commercial video soundtracks.** You must follow **each site’s terms** and, where applicable, **performance / sync rights** for published video.

**Cench Studio:** Background music in the editor is intended to come from **cleared sources** — today primarily **Pixabay Music** and **Freesound** via `POST /api/music/search` (see `lib/audio/providers/pixabay-music.ts`, `freesound-music.ts`), or **your own uploads** under terms you control. Use the Megathread as a **personal discovery bookmark list**, not as a built-in music library.
